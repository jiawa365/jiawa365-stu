1.0.8 alpha от 14.03.2002

   - Возвращена работоспособность под windows 95.

   - Изменено назначение имени выходного файла по умолчанию. Теперь
     имя файла формируется на основании текущей даты и времени.

   - VBR заработал с частотами дискретизации < 32 кГц.

   - Добавлена возможность использования фильтров высоких и низких
     частот.

   - Слегка изменен алгоритм конкурирования за ресурсы процессора.

   - Добавлена возможность временной приостановки записи (Ctrl + P).

   - Добавлен переменный битрейт (-vbr).

   - Добавлено воспроизведение того, что получается :) (-play).

   - Добавлена возможность нарезки файлов по паузам (-split4).
  
   - Добавлена возможность использовать  внешний mp3 acm кодек (-fhg).
     Кодек должен  быть установлен в системе,  для использования его с
     битрейтами > 96 кбит/с рекомендуется процессор не хуже iCel-600.

   - Добавлена возможность снимать сигнал со звуковой платы с частотой
     дискретизации 64 и 96 кГц :)

   - Добавлена возможность  удаления постоянной  составляющей сигнала.
     Этот режим включен по умолчанию. Отключается параметром -dc.

   - Исправлена ошибка,  иногда приводящая к перемене  местами стерео-
     каналов и/или смешиванию их :) при автоматическом вырезании пауз.

   - Добавлена возможность скрывать окно программы при работе (-hide).

   - Добавлена возможность использования average bitrate (ABR) (-abr).

   - Возвращена  возможность кодирования с нормальным качеством, кото-
     рой не было, начиная с версии 1.0.5.

   - Добавлен интерфейс  для получения  информации о  работе программы
     из других приложений (см. interproc.h, mmp3_monitor.cpp).

   - Добавлен вывод времени записи и имени выходного файла в заголовок
     окна.

   - Добавлена возможность указания времени ограничения записи в неце-
     лых значениях минут.

   - Исправлена ошибка  неправильного вывода времени ограничения запи-
     си.

   - Исправлена пара мелких багов.

1.0.7 от 13.11.2000

   - Исправлена  ошибка,  приводящая к небольшой  задержке при запуске
     программы, если указан ключ -a

   - Исправлена ошибка, иногда (редко) приводящая к остановке  записи.
     Ошибка была вызвана не точному следованию документации SDK.

   - Дополнена документация - отсутсвовало описание ключа -t.  Спасибо
     Егору Рокину <Egor@sibi.chita.ru>.

   - Добавлена  возможность завершения работы  программы  по появлению
     файла-флага. Файл-флаг при этом удаляется.

   - Введены возможные частоты дискретизации 8000, 11025, 12000 Гц.

   - Убран режим ms-stereo.

   - Убран фильтр низких частот.

   - Использованиа версия lame 3.87

   - Исключена предварительная  версия утилиты для разбивки  mp3-файла
     на части по паузам.

   - Добавлена  возможность  создавать новый mp3-файл каждый час.  Три
     режима работы: суточный, недельный, месячный.

   - По многочисленным просьбам сделан англоязычный интефейс. Извините
     за кривость :)

1.0.6 от 26.08.1999

   - Добавлена  возможность  выбора  частоты дискретизации из ряда 16,
     22, 24, 32, 44 или 48 кГц.

   - Расширен диапазон  поддерживаемых битрейтов, введена поддержка 8,
     16, 24, 32, 40, 48, 56, 144 кбит/c.

   - Добавлена возможность указания произвольной частоты среза высоких
     частот.

   - Исправлена   ошибка,   приводящая   к  некорректному   завершению
     получаемого mp3-файла.

   - Включена предварительная версия утилиты для разбивки mp3-файла на
     части по паузам.


1.0.5 от 08.08.1999

   - Добавлена возможность фильтрации записываемого сигнала на предмет
     срезания частот выше 11, 13 или 16 кГц.

   - Расширен диапазон поддерживаемых битрейтов, теперь поддерживаются
     64, 80, 96, 112, 128, 160, 192, 224, 256 и 320 кбит/с.

   - Добавлено  отображение  % использования   процессорного  времени.
     Работает только под Windows NT.

   - Слегка  изменен  механизм  "конкурентной борьбы"  за процессорное
     время.

   - Улучшена  стабильность  при очень активных  дисковых операциях со
     стороны других приложений.

   - Убрана  возможность  кодирования с нормальным качеством. Осталось
     низкое и высокое качество.

   - Изменен  алгоритм  определения  рабочей  полосы  частот.   Теперь
     ограничение   частотной   характеристики  сверху  можно  включать
     "вручную".  Раньше  автоматически  использовалось  ограничение по
     границе 16 кГц при битрейтах меньше 160 кбит/с.

   - Добавлено ожидание нажатия Enter после  вывода списка параметров.

   - Исправлена ошибка неправильного отображения на экране имен файлов
     с русскими буквами.

   - Дополнена документация.


1.0.4 от 01.08.1999

   - Добавлена возможность автоматического вырезания пауз из сигнала и
     указания порогового уровня для пауз.

   - Улучшен  механизм  "борьбы"  за  процессорное  время.  Теперь при
     уменьшении  остатка  буфера ниже  50%  приоритет  кодера временно
     повышается, пока остаток места в буфере не достигнет 55%.

   - Улучшен механизм  синхронизации потоков, и, как следствие, лучшее
     использование процессорного  времени в ходе конкуренции с другими
     приложениями. Пропуски звука получить становится все сложнее.


1.0.3 от 30.07.1999

   - Использовал компилятор от Microsoft.  Исполняемый код стал меньше
     и  быстрее.   Ранее  использовавшийся  Borland  C++  Builder  3 в
     сравнении с ним не выдерживает никакой критики.

   - Возможность  использования  движка  BladeEnc  убрана.  Lame более
     быстр и более функционален и качеством не хуже.

   - Добавлена   возможность  задания   режимов   кодирования  stereo,
     joint-stereo, mono. Раньше был толькько stereo.

   - Добавлена возможность указания качества кодирования - low, normal
     и high. Раньше было только high.

   - Программа заработала под Windows 98. Как там все криво...

   - Добавлена   возможность    устанавливать    повышенный  приоритет
     выполнения  кодера.  Иногда помогает в нелегкой борьбе за ресурсы
     центрального  процессора,  особенно  под  Windows  98.  Для  него
     возможность включается по умолчанию.

   - Внесены мелкие косметические исправления.


1.0.2 от 29.07.1999

   - Добавлена  возможность  выбора  используемого движка BladeEnc или
     Lame.

   - Добавлена  возможность  задания размера  буфера для неупакованных
     данных.

   - Изменен  алгоритм  задания  размера  буфера  по умолчанию. Теперь

   - Улучшена стабильность при работе с движком Lame. Ранее при выходе
     иногда программа вываливалась с ошибкой.

   - Внесены мелкие косметические исправления.
