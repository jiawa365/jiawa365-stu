1. Что такое MixMP3?

   MixMP3 - это кодер звуковой информации  в  популярный формат mp3 со
   звуковой  платы.  Если  быть  более  точным,  то это не полноценный
   кодер, а лишь надстройка над движком Lame или Fraunhofer acm codec.
   Программа  позволяет записывать звуковую информацию непосредственно
   со звуковой платы на жесткий диск в mp3-формате в реальном времени,
   если  конечно позволяют возможности аппаратуры. Программа выполнена
   в виде консольного приложения.


2. Возможности.

   - Запись звуковой информации в mp3-формат с битрейтом 8, 16, 20,
     24, 32, 40, 48, 56, 64, 80, 96, 112, 128, 144, 160, 192, 224, 256
     или 320 кбит/с; 16 бит; 8, 11, 12, 16, 22, 24, 32, 44 или 48 кГц,
     в режиме stereo, joint-stereo и mono. (см. ключи -b, -d, -m).
   - Автоматическое "выкусысание" пауз по заданному уровню (-s).
   - Автоматическая  резка  mp3  потока  на  файлы 4-мя различными
     способами, в том числе и по паузам (-split1..4).
   - Возможность выбора используемого энкодера - lame или fhg
     layer-III acm codec (должен быть установлен в системе) (-fhg).
   - Работа в режиме "шпиона", когда окно программы невидимо (-hide).
   - Запись с частотой дискретизации 64 или 96 кГц со звуковой платы
     (интересно, есть у кого-нибудь такие?:). При этом создается mp3
     файл(ы) с частотой дискретизации 48 кГц :) Только для lame.
   - Выбор качества кодирования - высокое, низкое, нормальное
     (среднее). Только для lame. (-ql, -qh).
   - Завершение работы по прошествии указанного времени. Время
     задается в минутах (может быть нецелое кол-во минут) (-t).
   - Установка повышенного приоритета для энкодера. Актуально при
     работе в Win9X (-a).
   - Выход по появлению заданного файла-флага (-x).
   - Автоматическое удаление постоянной составляющей сигнала
     (отключение -dc).
   - Использование режима lame - ABR (average bitrate) (-abr).
   - Использование режима lame - VBR (variable bitrate) (-vbr).
   - Возможность контроля за работой программы из других приложений
     (см. interproc.h, mmp3_monitor.cpp).
   - Высокая "конкурентноспособность" за ресурсы системы в многозадач-
     ном окружении.
   - Воспроизведение получаемого mp3-потока (-play).
   - Использование фильтров высоких и низких частоот (-fh, -fl).


3. Минимальные требования к системе.

   Операционая система:	     Windows NT 4, Windows 2000 или
                             Windows 9X/ME
   Процессор:		     iP-200 (для режима mono или низкого
                             качества) или iCeleron 333 (стерео с
                             высоким качеством)
                             для эффективного использования fhg
                             энкодера требуется процессор с тактовой
                             частотой не менее 600 МГц
   Оперативная память:	     32 Mb
   Звуковая карта:	     16-битная звуковая плата.


4. Краткое описание параметров командной строки.

   -b - битрейт, из ряда 8, 16, 24, 32, 40, 48, 56, 64, 80, 96, 112, 128,
        144, 160, 192, 224, 256, 320 кбит/c, по умолчанию 128 кбит/c
        в режиме vbr задает минимальный битрейт, по умолчанию 64 кбит/c
   -d - задать рабочую частоту дискретизации, из ряда 8, 11, 12, 16, 22,
        24, 32, 44, 48, 64 или 96 кГц, по умолчанию 44 кГц; если указана
        частота более 48 кГц, то она будет преобразована в 48 kHz
   -t - ограничение кодирования по времени в минутах
        по умолчанию - нет ограничения
   -f - размер буфера в кб для неупакованных данных, от 512 кб
        по умолчанию - 5%% от объема ОЗУ (для mono - 2.5%%)
   -s - вырезание пауз по указанному уровню (в dB)
        по умолчанию используется уровень -50 dB
   -a - установить повышенный приоритет выполнения кодера,
        по умолчанию для Windows 9x, -a- отменяет этот режим
   -x - задает имя файла-флага, по появлению которого программа
        завершает работу, файл-флаг при этом удаляется
   -dc - запрещает удаление постоянной составляющей сигнала
   -ms - использовать режим stereo (по умолчанию для битрейтов >= 192)
   -mj - использовать режим j-stereo (по умолчанию для битрейтов < 192)
   -mm - использовать режим mono
   -ql - кодировать с низким качеством
   -qh - кодировать с высоким качеством (по умолчанию - нормальное)
   -qv - кодировать с максимальным качеством
   -dr - запретить энкодеру изменять частоту дискретизации
   -fl - использовать фильтр низких частот
   -fh - использовать фильтр высоких частот
   -abr - включает использование режима ABR (average bitrate)
   -fhg - использовать fhg mp3 acm кодек (тестовый режим)
   -vbr n - использовать переменный битрейт (VBR), n - качество vbr (0..9)
   -hide - скрыть окно mixmp3, используйте файл stopmp3.flg для выхода
   -play - автоматически воспроизводить получаемый mp3-поток
   -devlist - вывести список устройств записи/воспроизведения
   -pdev - использовать при воспроизведении устройство n
   -rdev - использовать при записи устройство n
   -split1 - разбивать на файлы длительностью 1 час, суточный цикл
   -split2 - разбивать на файлы длительностью 1 час, недельный цикл
   -split3 - разбивать на файлы длительностью 1 час, месячный цикл
   -split4 - разбивать на файлы по паузам, см. так же ключ -s
